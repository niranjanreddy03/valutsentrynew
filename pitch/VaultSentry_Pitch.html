Detection Engine — Multi-Layer Architecture

Four detection layers work together for maximum coverage with minimal false positives.

🔴 Layer 1: Regex Pattern Matching

100+ curated patterns for all major cloud providers. AWS keys, Google API keys, Stripe tokens, GitHub PATs, private RSA/SSH keys, and database connection strings.

AWS (12)GCP (8)Azure (6)Stripe (4)GitHub (5)JWT (3)SSH Keys (6)DB URLs (8)

🟡 Layer 2: Entropy Analysis

Shannon entropy calculations detect high-randomness strings that may be secrets without matching known patterns. Threshold-based scoring with base64 and hex detection.

Shannon EntropyThreshold: 4.5Base64Hex Strings

🟣 Layer 3: Keyword & Context Scanning

Identifies assignment patterns like password=, api_key=, secret= in code and config. Context-aware: boosts for production, reduces for test files.

🟢 Layer 4: ML Risk Scoring

XGBoost/Random Forest models score findings with confidence. Considers environment, data classification, team ownership. Continuously retrains from user feedback.

Risk Classification Matrix

Critical (80-100)

AWS keys, private keys, production credentials

High (60-79)

API keys, database URLs, auth tokens

Medium (40-59)

Generic secrets, config values

Low (20-39)

Weak indicators, possible false positives
